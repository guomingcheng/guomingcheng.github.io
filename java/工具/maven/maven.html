# 简介

#
maven 是用来构建项目与简化 jar 包的管理

maven 是一款服务于 java 平台的自动化构建工具，对于一个大工程的项目使用 maven 来管理也是很方便的，因为 maven 可以创建多个子项目。maven 也统一了 jar 依赖。

# 生命周期

#
maven 三个独立的生命周期

# maven 定义了三个生命周期，又互不干扰，每个生命周期，都会分为几个步骤，顺序依次排序，越往后的环节执行时，都是总从第一个环节开始依次执行。
# 执行的每个步骤都会调用一个插件来处理，而这些插件就是实现了 maven 提高的生命周期的接口

#
了解 maven 自动化构建过程

# 在这个构建中的环节，会分为几个步骤，顺序依次排序，越往后的环节执行时，都是总从第一个环节开始依次执行。下面就是对构建中的环节进行介绍：

命令 翻译 说明
clean 清理 删除编译后的 class 字节码文件，为下次的编译环节做准备
compile 编译 自动下载依赖的jar包，裁决冲突的jar包，并将 java 源代码编程成 class 字节码文件
test 测试 自动测试，会自动的去调用 junit 编写的程序
@TableFi 报告 显示出测试的报告
package 打包 为当前项目打包，javaWeb 工程就打成 war 包, java 程序打成 jar 包
install 安装 maven 特定的概念，把打好的 jar 包复制道 “仓库” 中，参考是 maven 存放 jar 包的地方
deploy 部署 将动态 web 工程生成的 war 包负责到 servlet 容器指定的目录下，使其可以运行成功

# 默认的目录

#
使用 maven 创建工程默认的目录结构

# maven 要实现自动化构建，需要知道 java 源文件放在哪个目录，配置文件放在哪个目录，测试模块在哪个目录，要在这些已知的情况下才能实现构建。所以 maven 工程的创建，都有那些目录都是约定好的，目录分别要是做什么的，下面就是目录的结构与说明
|---holler        //工程名称         
|------scr              //源码
|---------mian               //主程序
|------------java                  //java 源文件
|------------resources             //存放 xml 已经的配置文件
|---------test               //测试程序
|------------java
|------------resources
|------pom.xml          //maven  工程的核心文件

# pom 文件

一个 maven 工程核心核心文件就是 pom.xml

pom 文件是 Project Object Model 项目对象模型的简写，一个工程构建的过程相关的配置都是在这个 pom 文件中进行配置，其重要程度相当于 WEB 工程的 web.xml 文件

#
pom 中的坐标

# 在maven仓库中查找是需要通过坐标的，而坐标分别使用 groupid 、artifactid、 version 这三个向量在仓库中定位一个工程
<?xml version="1.0" encoding="UTF-8"?>

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
 <modelVersion>4.0.0</modelVersion>

 <groupId>com.guomingcheng</groupId>               //公司名称或者组织域名 + 工程名称
 <artifactId>appointment</artifactId>            //模块或项目名称
 <version>1.0-SNAPSHOT</version>                      //版本
 
</project>

#
pom 中的依赖文件存放的仓库

# 仓库是保存 maven 自身所需要的插件、第三方框架或工具的 jar、与自己开发的 maven 工程。
在 pom 文件当使用坐标去引入依赖时， maven 会自动的从本地仓库查找并导入，如果本地仓库没有就去中央仓库去下载到本地在导入。

本地仓库

   (1)是当前电脑部署的 maven 仓库目录，服务与当下环境的 maven 工程


远程仓库

   (1)私服 : 是一个搭建在局域网的中央仓库，服务于当下的局域网环境的 maven 工程
   (2)中央仓库 ： 是一个搭建在全球网络中的中央仓库，服务于全国网络环境的 maven 工程
   (3)仓库镜像 ： 是一个搭建在一国网络中的仓库，用于分担中央仓库的并发能力

#
pom 中的依赖以及依赖范围

# 如在一个工程中需要依赖自己开发 jar 包，那么就需要在开发 jar 包哪个工程，使用 mvn install 命令把当前工程安装到 maven 的本地仓库，再使用坐标来引入当前工程中

# 依赖

#
依赖的版本之间冲突裁决

# 通过 maven 依赖的传递性，jar 包都是嵌套执行的引入，这容易出现一个问题，如果引入 n 个第三方 jar 包，这 n个要依赖了 guomingcheng.jar ,但互相依赖的版本又不同，这就会造成版本之间的冲突。而为了解决这方面的问题，maven 就有着 jar 引入的规则

优先级高：路径最短原则，知道 jar 包的依赖是嵌套的，把嵌套横向来看，哪个 jar 包距离 pom 文件最短，就使用哪个 优先级低：如俩者的距离都是一样，那么谁先声明就使用哪个

根据声明的原则，那么最终引入是 guomingcheng:1 版本

#
命令: mvn dependency:tree , 可以查看 maven 最终决策出来的 jar 包

#
依赖的排除

# 当导入第三方的依赖时，第三方依赖 jar 包也是一并导入的，那么有一个场景是，想要替换掉第三方的依赖，那么就要使用《依赖的排除》，把第三方的依赖先排除掉
<?xml version="1.0" encoding="UTF-8"?>

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
 <modelVersion>4.0.0</modelVersion>

 <groupId>com.guomingcheng</groupId>               //公司名称或者组织域名 + 工程名称
 <artifactId>appointment</artifactId>            //模块或项目名称
 <version>1.0-SNAPSHOT</version>                      //版本

 "依赖引入"
 <dependencys>
   <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>4.11</version>
      <scope>provided</scope>               "工程构建过程的依赖范围"
      <exclusions>              "排除父标签，可以有多个 exclusion"
          <exclusion>                     "排除标签，指定一个 jar 的坐标，版本号可以不指定，"
            <groupId>org.hamcrest</groupId>
            <artifactId>hamcrest-core</artifactId>
          </exclusion>
      </exclusions>
   </dependency>
 </dependencys>
 
</project>

#
依赖的版本统一管理

# 在 pom 中引入多个 jar 包时，版本号在今后是可能要修改的，所以要把所有的 jar 的版本号统一起来，便于今后维护 就需要使用 maven提供的 标签来声明，这个标签内可以声明自定义的标签，这样来管理 pom 文档中的版本号。
<?xml version="1.0" encoding="UTF-8"?>

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
 <modelVersion>4.0.0</modelVersion>

 <groupId>com.guomingcheng</groupId>               //公司名称或者组织域名 + 工程名称
 <artifactId>appointment</artifactId>            //模块或项目名称
 <version>1.0-SNAPSHOT</version>                      //版本

 <properties>  "声明版本号管理标签"
   <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
   <junit.version>4.11</junit.version>         "<junit.version> 就是自定义的标签，可以随意自定义"
 </properties>

 "依赖引入"
 <dependencys>
   <dependency>
     <groupId>junit</groupId>
     <artifactId>junit</artifactId>
     <version>${junit.version}</version>       "使用 ${} 来引入声明版本号中的值"
   </dependency>
 </dependencys>
 
</project>

# 继承与聚合

#
管理多个子工程 jar 包的版本统一性

# 在 maven 中继承的作用便是用来管理多个子工程 jar 版本的统一，在我们创建一个工程时，是要声明 jar 或者 war 工程，但是还可以声明为 pom 工程，当声明为 pom 时，表示当前工程是用来管理子工程 jar 包的

注：除了以上是三种工程，还可以声明为 maven-plugin ，表明当前工程是 maven 插件

# 父工程

<?xml version="1.0" encoding="UTF-8"?>

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
 <modelVersion>4.0.0</modelVersion>

	<groupId>com.guomingcheng</groupId>
	<artifactId>HelloFriend</artifactId>
	<version>1.0-SNAPSHOT</version>
	<packaging>pom</packaging>                  "声明当前为 pom 工程"

	<modules>     "聚合标签，当安装父工程时，也一并安装用 modules 配中的工程"
        <module>../HelloTest</module>
    </modules>

	<properties>
		<junit.version>4.11</junit.version>
		<servlet-api.version>3.1.0</servlet-api.version>
	</properties>

	<dependencyManagement>   "管理子工程的 jar 包，需要用当前标签包裹着 dependencies 标签就行，其他的没有改变"

		<dependencies>
			<dependency>
				<groupId>junit</groupId>
				<artifactId>junit</artifactId>
				<version>${junit.version}</version>
			</dependency>

			<dependency>
				<groupId>javax.servlet</groupId>
				<artifactId>javax.servlet-api</artifactId>
				<version>${servlet-api.version}</version>
			</dependency>
		</dependencies>

	</dependencyManagement>

</project>

# 继承父工程，当前为子工程

子工程的继承，可以继承版本号，也继承了依赖范围。
注意一，子工程的依赖也使用 version scope 这俩个标签，那么它的优先级会比父工程高
注意二，如果要 install 子工程时，先必须 install 父工程，否则 install 时是找不到父工程的

<?xml version="1.0" encoding="UTF-8"?>

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
 <modelVersion>4.0.0</modelVersion>

 <groupId>com.guomingcheng</groupId>
 <artifactId>HellWar</artifactId>
 <version>1.0-SNAPSHOT</version>
 <packaging>war</packaging>

 <parent>            "parent 是继承标签，映入父工程的坐标"
   <groupId>com.guomingcheng</groupId>
   <artifactId>HelloFriend</artifactId>
   <version>1.0-SNAPSHOT</version>

   <relativePath>../HelloFriend/pom.xml</relativePath>  "这是声明父工程的 pom.xml 文件的路径"
 </parent>

 <properties>
   <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
 </properties>

 <dependencies>
   "使用依赖，不需要使用版本号标签，依赖版本会继承父工程，以及继承 jar 包依赖的范围"
   <dependency>        
     <groupId>junit</groupId>
     <artifactId>junit</artifactId>
   </dependency>

   <dependency>
     <groupId>javax.servlet</groupId>
     <artifactId>javax.servlet-api</artifactId>
   </dependency>

 </dependencies>

</project>

# 插件

#
一个个插件是可以绑定多个周期阶段的

# 一个插件可以包含多个目标方法，而一个目标方法只能绑定在一个生命周期阶段，否则会出现警告。
# 自定义的插件绑定生命周期某个阶段，并不能覆盖掉 maven 默认的插件，而是俩者并行存在。

# 自定义插件

# 下面导入的 jar 包是用来实现 maven 插件的 api
<?xml version="1.0" encoding="UTF-8"?>

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
 <modelVersion>4.0.0</modelVersion>

 <groupId>com.guomingcheng</groupId>
 <artifactId>HelloPlugin</artifactId>
 <version>1.0-SNAPSHOT</version>

 <packaging>maven-plugin</packaging>       "当前工程声明为 maven-plugin"

 <properties>
   <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
   <maven.compiler.source>1.8</maven.compiler.source>
   <maven.compiler.target>1.8</maven.compiler.target>
 </properties>

 <dependencies>

   <dependency>
     <groupId>org.apache.maven</groupId>
     <artifactId>maven-plugin-api</artifactId>
     <version>3.0</version>
   </dependency>

   <!-- maven 插件的 注解 -->
   <dependency>
     <groupId>org.apache.maven.plugin-tools</groupId>
     <artifactId>maven-plugin-annotations</artifactId>
     <version>3.4</version>
     <scope>provided</scope>
   </dependency>

 </dependencies>

</project>

# 插件代码编写

//指定目标方法
@Mojo(name = "log")
public class MyHelloPlugin extends AbstractMojo {

   // Parameter 是用来传递参数的
   // property 的值是在 pom 文件中配置的变量，通过把值映射过来。defaultValue 表示用户没有传值那么默认把 Peter 映射给 name log
   @Parameter(property = "log.name", defaultValue = "Peter")
   private Object name;

   @Override
   public void execute() throws MojoExecutionException, MojoFailureException {
       getLog().info("Holle guomingcheng " + this.name );
   }
}

# 命令行执行插件

# 命令： mvn com.guomingcheng:HelloPlugin:1.0-SNAPSHOT:log -D log.name=百度
# 解释： mvn [groupId]:[artifactId]:[version]:[目标方法名称] -D [变量]=[值]
# 解释： -D 是可选的

# pom.xml 绑定插件

<?xml version="1.0" encoding="UTF-8"?>

<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">

 <packaging>jar</packaging>

 <properties>
   <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
   <maven.compiler.source>1.7</maven.compiler.source>
   <maven.compiler.target>1.7</maven.compiler.target>
 </properties>

 <dependencies>

 </dependencies>

 <build>
   <plugins>

     <plugin>
       <groupId>com.guomingcheng</groupId>        "自定义插件的坐标"
       <artifactId>HelloPlugin</artifactId>
       <version>1.0-SNAPSHOT</version>
       <executions>        "绑定配置多个生命周期某个阶段"
         <execution>
           <phase>pre-clean</phase>     "绑定哪个生命周期阶段"
           <goals>
             <goal>log</goal>    "调用目标方法"
           </goals>
         </execution>
         <configuration>  "配置参数传递，仅对当前目标方法生效"
             <name>baidu</name>
         </configuration>  
       </executions>
       <configuration>    "配置参数传递，对当前坐标插件所有目标方法都生效，可以传入多个。前缀的 log, 可以不用。<name> 就代表着了"
         <name>com.guomingcheng.cn</name>
       </configuration>
     </plugin>

   </plugins>

 </build>

</project>
