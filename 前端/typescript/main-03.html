# 函数类型表达式

这 ts 中，函数也是一种类型，具备了函数头以及返回值，他是没有实现函数体，这个需要注意

// 这个 fun 变量定义一个函数类型，并且约束了这个函数的类型参数是 (s: string) 以及返回值 void, 
// 如果一个函数等价的满足就无法赋值给 fun 变量
let fun: (s: string) => void 

// 如果一个函数实现了函数体，那么就表示这个不是函数类型了，他可以称为函数值
function pinTo(a: string){
    console.log(a)
}

// pinTo 函数值，具备 fun 变量的约束，所以把 pinTo 函数值赋给 fun 是没有问题的
fun = pinTo
fun("a")

# 函数签名

这个函数签名不仅拥有函数的特性，还可以让函数有自己的变量，ER6 函数表达式是不具备这个特性的，这个要注意

// 为这个类型取一个别名，这个是函数类型，并且拥有自己的属性 describable
type DescribableFunction = {
    describable: string,
    // 可以这样理解，这个函数类型并没有绑定一个变量，fn() 这样直接找到这个没有绑定变量的函数类型
    (someArg: number): boolean
}

// 这个 fn 是一个函数类型
function doSomething(fn: DescribableFunction) {
    // fn 调用自己的属性 describable     
                    // 这里我很不解，上面明明是一个对象的类型，为何这里可以直接使用 《函数调用》
    console.log(fn.describable + " returned " + fn(1))
}

function fn1(n: number): boolean{
    return true;
}
// 绑定函数的属性，
fn1.describable = "guomingcheng"

doSomething(fn1) // 调用

# 泛型函数

这个是 java 的泛型一样，用于约束一个函数参数以及返回值都是一个类型

// 这是定义了一个泛型函数，根据 firstElement() 方法传入的参数自动的推断出 T 是什么类型
function firstElement<T>(arr: T[]): T|undefined{
    return arr[0]
}

// 比如这里调用了 firstElement() 函数，传入的是一个 number 数组实列，那么泛型会地址的把 T 替换成 number 类型
// 这个写法，Ts 会根据传入的参数推断出来 T 的类型
firstElement([1,2,3])

// 这个写法，是约束了这次传入的参数必须是 string 类型
firstElement<string>(["1","1"])

# 泛型约束条件

在 ts 的泛型中，也如同 any 类型类似，如果你要在函数的内部调用参数的方法，这样就出现编译错误，因为泛型是无法知道用户调用这个方法会传入什么参数 ，所以，我们要使用与 java 类似的泛型边界，编译器才能知道这个泛型不能传入超过边界的类型，编译才不会出现错误


// 这里的泛型函数就使用了边界，他这个表示，传入的参数类型必须具备了 lethth:number 属性，比如数组类型、string 类型都是具备了这个属性

// 个人认为，他这个写法以及体现不出泛型的好处，可能在 class 才能有泛型的用武之地
function longest<T extends {letgth:number}>(a: T, b: T){
    if(a.letgth > b.letgth){
        return 0;
    }
}

// 函数调用，看下面就知道，letgth 属性是一定要具备了，可以在基础上添加额外的属性也不会报错，因为在函数中并没有调用
longest({letgth: 1}, {letgth: 3, nm: 3})

# 函数的可选参数

在 Ts 中，函数的参数增加了俩个特性，分别是可选的以及默认值


// 这个表示，sum 参数是一个可选的，可以选择不传也可以选择传
function game(sum?: number){}

// 这个表示，如果用户传入参数，sum 的值默认是 100
function game(sum: number = 100){}

# 函数重载

在 Ts 中的函数重载感觉非常尧，不明白作用体现在什么地方，因为他这个重载的发送与设置可选参数，区别有和不同，下面只是记录重载的实现方式：

// 这是 Ts 的重载方式，相同的函数名称，不同的参数个数，以及相同的返回，就是重载，下面俩行只是定义了要重载函数的接口
function makeDate(timestamp: number): Date
function makeDate(m: number, d: number, y: number): Date

// 重载实现
// 在这个重载实现中，必须是具备了上面函数接口中的参数类型兼容，以及参数的个数兼容，以及返回值的类型兼容，这三具备了才能实现重载
function makeDate(mOrTimeStamp: number, d?: number, y?: number): Date{
    if(d != undefined && y != undefined){
        return new Date(y, mOrTimeStamp, d);
    }else{
        return new Date(mOrTimeStamp);
    }
}

// 调用重载函数需要注意的一点是，这里是看不懂到《重载实现参数列表的约束的》，所看到是函数接口参数的约束
// 调用 makeDate 函数可以传入一个参数或者三个参数，都是可以，但是不能传入2个参数，因为 makeDate 函数接口并没有定义了 2 个参数
makeDate(1110)

# 了解其他类型

# void

在一个函数没有设置返回值
或者在函数体中没有返回值的时候，Ts 会自动的推断返回值是 void , void 表示是空或者是没有的意思

# object

object 指的是任何不是基元类型, 不知道他这个是否也包含了 class 
而基元类型的是:
string number bigint  boolen symbol null undefind

object 是不同于 {} ，不同于 Object , 小写的 object 与这俩个是不等价的，在 TS 中要使用小写的 object

# unkknown

unknown 类型代表着任何的值，这与 any 类型类似，但是更为安全，因为对于未知的 unknown 类型的值做任何事情都是非法的

// 这里使用 any 类型， 调用未知的 b() 函数，对于 any 是合法通过的，但是并不安全
function fn1(a: any){
    a.b();
}


// 这里是不合法的，因为无法知道用户传入的是什么类型的值，unknown 更为严格的限制，这比 any 安全
// 如果把 unknown 类型作为返回值的话，只能作为只读变量，因为这个变量是否是有属性对于 unknown 类型都是未知的，调用就是不合法
function fn2(a: unknown){
   // a.b()
}

# 可变参数

他这个可变参数与 java 的可变参数都是一样，只不过定义的方式有些不一样

// 变量前面 + ... , 这样就是可变参数的定义，不同点至于类型必须是一个数组，这是需要注意
// number[] .
function multiply(...m: number[]){
    m[0]
}

// 这样，我们就可以传入任意个 number 类型的参数，他这个可变参数就是把我们传入的 number 何必到一个 number[] 赋值给 m
// 在 Ts 中，叫做形参展开
multiply(1,2,3)

// 实参展开
let arr1 = [1 ,2 ,3]
let arr2 = [1, 3, 4]

// 这个就是实参展开的实现，这样意思就是把 [1,2,3] 变成 1，2，3 传递到 map() 中
arr1.push(...arr2)

# 形参解构

形参解构就是一个把对象的属性分解出来，通常函数内访问对象类型中的属性，需要通过 变量名.属性 ，这个方式访问。如果采用解构模式，访问属性只通过属性名称即可

// 为这个对象类型取一个别名
// b 属性是一个可选的
type Abc = {a: number, b?: number, c: number}

// 这个就是形参编写方式，他把对象类型内的属性分解给了左边的变量
// 如果传入的对象没有 b 属性，那么 b 默认值就是 0
function sum({a,b = 0,c}: Abc){
    
    // 这样就可以直接访问对象内的属性
    console.log(a + b + c)
}
