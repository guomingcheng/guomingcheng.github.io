# groph 启动

Groph 节点启动命令:

docker run -it -e postgres_host=host.docker.internal
               -e postgres_port=5432     
               -e postgres_user=postgres 
               -e postgres_pass=xxxxxxxxxx
               -e postgres_db=graphnode    
               -e ipfs=host.docker.internal:5001 
               -e ethereum=mainnet:http://host.docker.internal:8545/ graphprotocol/graph-node:e519812     

docker run -it -e postgres_host=host.docker.internal -e postgres_port=5432 -e postgres_user=postgres -e postgres_pass=gmC24354. -e postgres_db=graphnode -e ipfs=host.docker.internal:5001 -e ethereum=mainnet:http://host.docker.internal:8545/ graphprotocol/graph-node:e519812

当出现这个问题，是因为 ganache 节点的数据是内存操作的，每次关机数据都会丢失。当 graph 启动时检查链接的 RPC
与上个版本不一致，就会爆出区块错误。只需要 graph 重新链接一个 postgresql 一个新的数据库即可
